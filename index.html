Session history

How this works: problems are marked solved automatically — there's no manual checkbox to click in this build. Click 🔗 Connect LeetCode Profile at the top once (just your public username, no password), then use 🎓 Certificates → 🔄 Sync from LeetCode — it pulls a deep slice of your accepted-submission history (hundreds of solves, not just the last 20) and auto-checks any matches here. There's no real sign-in or live account access possible from a plain webpage, and LeetCode doesn't publicly expose a truly complete solved-problem list without your own session — so an extremely long history can still have a few very old solves fall outside what one pull returns, with no manual way to check those off yet.

Badges: ⭐ Top 150 means this problem is on LeetCode's official Top Interview 150 — whether it was already in the Striver A2Z sheet or added purely because of the official list, it gets the same badge, since it's Top 150 either way. Rows added into their matching topic section purely because of the official list still get a green left border so you can see what was newly added, just without a different badge. 🏢 Company (also a green left border) means it wasn't in the original sheet and was added purely because it's a frequently-asked problem at companies hiring in India. 🧊 NeetCode 150 and 🎯 Blind 75 mean the problem is on those respective lists — every single problem on both lists was already present in this tracker, so matching them added no new rows, just badges. 📗 Babbar 105 means the problem was matched to an entry on Love Babbar's 450 GeeksforGeeks sheet; most of that sheet is GFG-specific implementation and theory questions (build-a-stack-from-scratch, SPOJ-only problems, "why is X true" questions) with no single corresponding LeetCode problem, so only entries with a genuine LeetCode equivalent are tagged — about 105 of the sheet's ~445 listed problems.

🏢 Company tags: every problem shows the companies (up to 4, click "+N" for the full list) known to have asked it, per the community-maintained leetcode-company-wise-problems repo's full all-time lists, filtered to 58 companies with a major India hiring presence. Problems tagged only with Shell, Concurrency, or JS-utility categories were left out as out of scope for a DSA tracker; a few problems added purely from company tags don't have a known official LeetCode number, so they show a 🏢 badge instead of "#123". Use the company dropdown in the controls, or click "🏢 Companies" to browse a searchable grid of every tagged company with counts and jump straight to its problems. This is a point-in-time snapshot of that repo (pulled July 2026) — it won't auto-update if the repo changes.

New per-problem tools: 📝 opens notes/code-snippet + tags for that problem. ⏱ starts/stops a manual timer that logs time spent and counts an attempt (a page reload while it's running will lose that one in-progress interval, so stop it before closing the tab) — this feeds the streak, avg-solve-time, and weak-topic numbers in 📊 Analytics. 💡 gives an optional AI hint if you've added your own Gemini API key (Settings below) — the key is stored only in your browser's local storage and is sent directly from your browser to Google, never to us. Inside that same hint popup, 💻 Get the code expands into brute-force and optimal tabs with real runnable code, complexity, and an animated step-by-step visual walkthrough.

📅 Review Queue: when you mark something solved, it's scheduled to resurface for review after 3, then 7, then 14, then 30, then 90 days — standard spaced repetition. Click the toggle above to see what's due.

📊 Analytics: streak, time spent, solves-per-day, weakest topics by completion rate, and a solve-time trend — all computed from what's stored in this browser or your account (see "Where your progress lives" below).

🎤 Mock Interview Mode: pick a duration, difficulty, topic, and/or company, then get a randomly selected problem, a countdown timer, and a real code editor (any language — it's just a labeled text editor, not a compiler or judge). Self-grade when you finish or time runs out; every session is logged to Session History and feeds the Analytics tab's mock-interview stats.

Where your progress lives: this tracker stores everything — solved problems, notes, timers, mock-interview history — in your browser's local storage. It persists on this device/browser but won't follow you to another one automatically. Use Export Backup to save a JSON file and Import Backup to restore it on another device or browser.

        ⚙️ Background sync via GitHub Actions
        

          
This is the only sync method this tracker has — problems are checked off automatically, on a schedule, with no manual clicking. It works by putting your LeetCode session cookie into a repo you own as a GitHub Actions secret. That's a real trust decision, not a free upgrade — read this before setting it up:

          
    
            
  • Your session cookie is your LeetCode login — anyone who reads it can act as you (submit code, change your profile, see your email).
    • 
            
  • It never touches this tracker or any server of mine. It lives only in your own repo's encrypted Actions secrets, and is only decrypted inside your own workflow runs.
    • 
            
  • The repo needs to be public for this tracker to read data/solved.json back (anonymous fetch, no token on this end) — so your solved-slug list is publicly visible, similar to your public LeetCode profile. The secret itself stays encrypted either way.
    • 
            
  • Rotate the secret periodically — LeetCode session cookies expire on their own eventually, so you'll need to refresh it every so often regardless.
    • 
          

          
One-time setup (in your own GitHub account):

          
    
            
  1. Create a new public GitHub repo.
    2. 
            
  2. Download the two files below and add them to that repo at the exact paths named in the filename: .github/workflows/leetcode-sync.yml and scripts/leetcode-sync.mjs.
    3. 
            
  3. In the repo → Settings → Secrets and variables → Actions, add a secret named LEETCODE_SESSION with the value of your LEETCODE_SESSION cookie from leetcode.com (DevTools → Application → Cookies), and optionally LEETCODE_CSRF_TOKEN from the csrftoken cookie.
    4. 
            
  4. In the repo → Settings → Actions → General → Workflow permissions, set "Read and write permissions" so the workflow can commit the synced file back.
    5. 
            
  5. Run the workflow once manually (Actions tab → "Sync LeetCode solved list" → Run workflow) to confirm it writes data/solved.json. After that it runs automatically every 6 hours.
    6.
